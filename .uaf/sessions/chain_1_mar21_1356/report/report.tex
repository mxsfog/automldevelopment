\section{Executive Summary}

\subsection{Executive Summary}

\subsection{Цель}
Предсказание победы ставки (won/lost) на спортивных событиях. Задача — достичь ROI >= 10\% на отобранных моделью ставках при использовании flat betting.

\subsection{Лучший результат}
\begin{itemize}
  \item Метрика ROI: \textbf{+16.02\%} (тест)
  \item Стратегия: 3-model ensemble (CatBoost + LightGBM + LogReg) обученный на полном train, отбор ставок по Expected Value >= 0.12
  \item Объем ставок: 2247 из 14899 тестовых (15.1\%)
  \item AUC: 0.784
  \item Win rate: 36.6\%, средний коэффициент: 33.9
  \item Cross-validation: mean ROI = 18.73\%, std = 14.14\% (5 фолдов)
\end{itemize}

\subsection{Ключевые выводы}
\begin{itemize}
  \item EV-based selection (EV = model\_prob * odds - 1) — ключевой прорыв. Переход от probability threshold к EV threshold увеличил ROI с +1.5\% до +16\%
  \item Из 5 групп feature engineering только Sport/Market target encoding дал значимый прирост (+6.92 delta ROI). Остальные группы (odds-based, temporal, interactions, complexity) добавляли шум
  \item ROI обусловлен высокими коэффициентами (avg\_odds=33.9). При ограничении odds<=5 ROI падает до +2.6\%. Стратегия работает за счет выявления value в high-odds ставках
\end{itemize}

\subsection{Рекомендации}
\begin{enumerate}
  \item Для продакшена использовать EV>=0.12 без ограничения odds, но с мониторингом drawdown
  \item Высокая дисперсия (CV std=14.14\%) требует bankroll management (Kelly criterion или фиксированный процент)
  \item Периодическая переобучение модели на свежих данных (данные покрывают 81 день)
  \item Исследовать калибровку вероятностей для более точного EV-расчета
\end{enumerate}

\section{Analysis and Findings}

\subsection{Analysis and Findings}

\subsection{Baseline Performance}
\begin{itemize}
  \item DummyClassifier (все ставки): ROI = -1.96\% (n=14899)
  \item Случайный отбор 50\%: ROI = -5.92\% (n=7363)
  \item Правило ML\_P\_Model>=60: ROI = +0.90\% (n=4353) — первый положительный ROI
  \item LogisticRegression (thr=0.65): ROI = +1.59\% (n=5524), AUC=0.7918
  \item CatBoost default (thr=0.60): ROI = +0.80\% (n=5697), AUC=0.7934
\end{itemize}

Все baseline-модели дают положительный, но слабый ROI (< 2\%). Модели фильтруют в основном low-odds/high-probability ставки (win\_rate > 90\%, avg\_odds ~ 1.07).

\subsection{Feature Engineering Results}
Метод Shadow Feature Trick: обучение двух моделей (baseline vs candidate) с одинаковыми гиперпараметрами.

| Группа фич | Delta ROI | Статус |
|------------|-----------|--------|
| Odds-based (implied\_prob, margin и др.) | -0.10 | rejected |
| Sport/Market target encoding | +6.92 | accepted |
| Temporal (hour, day\_of\_week) | -6.25 | rejected |
| ML interactions (edge\_x\_odds и др.) | -5.68 | rejected |
| Complexity (odds\_spread, odds\_cv) | -6.89 | rejected |

Только 4 фичи Sport/Market encoding прошли проверку. Добавление любых других групп ухудшало результат. Вероятная причина: избыточность (Odds уже несет большую часть информации), шум на малых выборках.

\subsection{Model Comparison}
\begin{itemize}
  \item CatBoost: основная модель, AUC~0.79, Odds доминирует (51.6\% importance)
  \item LightGBM: сравнимый AUC, добавляет diversity в ансамбль
  \item LogisticRegression: слабее по AUC, но стабилизирует ансамбль
  \item XGBoost: добавление 4-й модели + дополнительных фич ухудшило ROI (-3.07\% vs +7.82\%)
\end{itemize}

3-model ensemble (CB+LGBM+LR) — оптимальный вариант. Добавление моделей или фич = переобучение.

\subsection{Segment Analysis}
\begin{itemize}
  \item Dota2: +4.58\% ROI (segment-level)
  \item Cricket: +4.13\% ROI
  \item CS2: +2.54\% ROI
  \item Ice Hockey, Basketball: отрицательный ROI на val
\end{itemize}

Попытка исключить "плохие" виды спорта на val привела к ROI -18.07\% на test. Сегментный анализ на val не переносится — слишком мало данных для надежной сегментации.

\subsection{Stability \& Validity}
5-fold time series CV для лучшей конфигурации (EV>=0.12, full train):

| Fold | ROI | N bets |
|------|-----|--------|
| 0 | +31.90\% | 1921 |
| 1 | +12.45\% | 2463 |
| 2 | +38.03\% | 1952 |
| 3 | +11.59\% | 1924 |
| 4 | -0.32\% | 1893 |
| \textbf{Mean} | \textbf{+18.73\%} | - |
| \textbf{Std} | \textbf{14.14\%} | - |

4 из 5 фолдов прибыльны. Fold 4 (последний по времени) почти на нуле — возможный drift.

Anti-leakage проверки:
\begin{itemize}
  \item Threshold (EV>=0.12) выбран на val, применен к test один раз
  \item Target encoding fit только на train
  \item Нет future-looking фичей (Payout\_USD, Score удалены)
  \item Test ROI 16.02\% < 35\% sanity threshold
\end{itemize}

Замечание: ROI обусловлен высокими коэффициентами. При odds cap <= 5 ROI падает до ~2.6\%. Стратегия прибыльна за счет выявления value в высоких коэффициентах, что несет высокий риск.

\subsection{What Didn't Work}
\begin{enumerate}
  \item \textbf{Probability threshold optimization} (Step 4.1): модель выбирает только safe bets с очень низкими коэффициентами (avg\_odds=1.06). ROI = -0.99\%
  \item \textbf{Sport segmentation} (Step 4.3): фильтрация по видам спорта на val дала ROI -18.07\% на test. Overfitting на малых сегментах
  \item \textbf{4-model ensemble + extra features} (Step 4.4): XGBoost + odds\_features + interaction\_features = ROI -3.07\%. Дополнительные фичи = шум, ухудшение EV-калибровки
  \item \textbf{Odds capping} (Step 4.6): ограничение max\_odds убивает ROI. При odds<=5 лучший ROI = 6.48\% (EV>=0.25, n=361). Alpha находится именно в high-odds ставках
  \item \textbf{Optuna hyperparameter tuning} (Step 3.1): минимальный прирост (+1.47\% vs +1.31\%). Гиперпараметры не критичны для данной задачи
\end{enumerate}
