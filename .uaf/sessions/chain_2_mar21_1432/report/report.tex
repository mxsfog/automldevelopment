\section{Analysis and Findings}

\subsection{Analysis and Findings}

\subsection{Baseline Performance}
Верифицированный baseline из chain\_1: ROI=+16.02\%, AUC=0.784. Воспроизведён с delta=0.00\% (step 4.0). Стратегия: 3-model ensemble (CatBoost + LightGBM + LogReg), EV threshold >= 0.12, full train.

\subsection{Feature Engineering Results}
19 принятых фичей из предыдущей сессии (15 базовых + 4 sport/market encoding). В этой сессии новые фичи не тестировались — фокус был на архитектуре и selection strategy.

\subsection{Segment Analysis}
\begin{itemize}
  \item Прибыль концентрирована в high-odds сегменте (avg\_odds=33.9)
  \item Low-odds ставки (odds<=5) дают ROI=+2.6\% — мало value
  \item Sport/market фильтрация ухудшает результат (chain\_1 step 4.3)
  \item Kelly criterion перекладывает вес с high-odds на low-odds, уничтожая ROI
\end{itemize}

\subsection{Stability \& Validity}
\begin{itemize}
  \item 5-fold time series CV: mean=18.73\%, std=14.14\%
  \item 4/5 фолдов прибыльны, fold 4 ~ 0\%
  \item Нет leakage: threshold выбран на val, применён к test один раз
  \item Stratified EV (ROI=52.94\%) — false positive (overfitting к val noise), не принят
  \item Profit regression val ROI=157\% вызвал MQ-LEAKAGE-SUSPECT alert (false positive)
\end{itemize}

\subsection{What Didn't Work}

\subsubsection{1. Калибровка вероятностей (step 4.1)}
Isotonic prefit калибровка размывала вероятности, EV порог падал с 0.12 до 0.11, захватывая больше шумных ставок.

\subsubsection{2. Более сложные модели (step 4.2)}
Optuna-тюнированный CatBoost (depth=8, iter=873) даёт менее калиброванные вероятности для EV-расчёта. Простая модель (depth=6, iter=200) оптимальна для этой задачи.

\subsubsection{3. Kelly criterion (step 4.3)}
Kelly stake = EV/(odds-1) пропорционально уменьшает ставку на high-odds. Но именно high-odds — источник прибыли. Kelly оптимизирует log-wealth, а не ROI.

\subsubsection{4. Profit regression (step 4.4)}
Target profit = won*odds - 1 имеет extreme skew (min=-1, max=+126). Модели не могут обучиться на таком распределении — даже Huber loss не спасает. Classification + post-hoc EV принципиально лучше, т.к. разделяет оценку вероятности и расчёт profit.

\subsubsection{5. Val-optimized thresholds}
Любая оптимизация порогов на val (stratified EV, weight search) не переносится на test из-за distribution shift между val и test периодами.
