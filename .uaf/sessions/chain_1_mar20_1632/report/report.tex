\section{Executive Summary}

\subsection{Executive Summary}

\subsection{Цель}
Построить ML-модель предсказания исходов ставок (won/lost) на спортивных событиях для отбора ставок с ROI >= 10%. Данные: 74 493 ставки (синглы и парлаи) по 10+ видам спорта, период декабрь 2025 -- февраль 2026.

\subsection{Лучший результат}
\begin{itemize}
  \item Метрика ROI: \textbf{7.26%} (val-selected threshold, anti-leakage compliant)
  \item Стратегия: Ensemble (CatBoost + LightGBM + XGBoost), средние вероятности, порог 0.60, исключение убыточных спортов (Basketball, MMA, FIFA, Snooker)
  \item AUC: 0.8097
  \item Объем ставок: 5 146 из 12 118 (42.5% отфильтрованного test set)
  \item Win rate: ~76%
\end{itemize}

\subsection{Ключевые выводы}
\begin{itemize}
  \item Segment filtering (исключение 4 убыточных спортов) дал наибольший прирост ROI: с 3.20% до 7.23% (+4.03 п.п.)
  \item Odds/implied_prob доминируют в feature importance (85%+). Модель преимущественно обучается на структуре коэффициентов
  \item Потолок ROI при текущих данных: ~7.2%, подтверждено 8 итерациями Phase 4 без существенного улучшения
  \item Target 10% не достигнут. Разрыв ~2.7 п.п. требует принципиально новых данных или подходов
\end{itemize}

\subsection{Рекомендации}
1. \textbf{Обогащение данных}: добавить внешние данные (ELO, form, head-to-head) -- текущие ML_Team_Stats_Found всегда 'f'
2. \textbf{Расширение временного окна}: 3 дня test set -- слишком мало для надежной оценки стабильности
3. \textbf{Динамическое исключение спортов}: автоматический мониторинг ROI по спортам с rolling window
4. \textbf{Online learning}: адаптация порога и весов ансамбля на свежих данных

\section{Analysis and Findings}

\subsection{Analysis and Findings}

\subsection{Baseline Performance}
Без ML все ставки дают ROI = -3.07% (bet_all baseline). Random selection: -5.97%. Правило ML_Edge >= 8 показало val ROI=15% vs test ROI=-3.53% -- классический overfitting. LogisticRegression: ROI=-1.40%, AUC=0.7913. CatBoost default: ROI=-2.86%, AUC=0.7938. Все baseline отрицательные -- задача нетривиальная.

\subsection{Feature Engineering Results}

\subsubsection{Что помогло}
\begin{itemize}
  \item \textbf{Odds-derived features} (log_odds, implied_prob, value_ratio): приняты через shadow feature trick, delta ROI = +2.05%
  \item \textbf{Interaction features} (edge_x_ev, model_implied_diff): улучшают discrimination
  \item \textbf{Parlay complexity} (Outcomes_Count * Is_Parlay): полезна для разделения типов ставок
\end{itemize}

\subsubsection{Что не помогло}
\begin{itemize}
  \item \textbf{Target encoding} (sport_winrate, market_winrate): REJECTED, delta ROI = -1.09%. market_winrate доминировала в importance (50%) -- утечка
  \item \textbf{Temporal features} (hour, day_of_week, is_weekend): REJECTED, delta ROI = -3.44%. Шумные и не несут сигнала для ROI
  \item \textbf{Kelly fraction, odds_squared, model_confidence}: REJECTED, не улучшают модель
  \item \textbf{Sport как categorical feature}: маргинальный эффект +0.03%, по сути бесполезно
\end{itemize}

\subsubsection{Feature Importance}
Из CatBoost feature importance: implied_prob (31%), Odds (28%), log_odds (26%), ML_P_Implied (11%). Остальные фичи -- менее 1% каждая. Модель преимущественно работает с odds-структурой.

\subsection{Model Comparison}

| Модель | ROI (filtered) | AUC | Threshold | Notes |
|--------|---------------|-----|-----------|-------|
| CatBoost single | 2.66% | 0.7945 | 0.65 | После Optuna |
| Ensemble avg (CB+LGB+XGB) | 7.23% | 0.8095 | 0.60 | Лучший подход |
| Weighted ensemble (0.7/0.15/0.15) | 3.13% | 0.7939 | 0.60 | Overfitting весов |

Equal average ensemble стабильно лучше любых весов и stacking. Причина: на шумных данных простое усреднение более робастно.

Лучший сегмент: Odds [1.5, 1.8) -- ROI=15.57% при n=1338. Но использование per-odds-range пороговOptimization переобучается на val и не переносится на test.

\subsubsection{По типу ставки}
Singles (Is_Parlay='f') стабильнее parlays, но разница в рамках фильтрованного набора невелика.

\subsection{Stability & Validity}

\subsubsection{Anti-Leakage}
\begin{itemize}
  \item Threshold выбирается на val (последние 20% train), применяется к test однократно
  \item Target encoding отвергнут из-за data leakage
  \item Time-series split исключает look-ahead bias
  \item Все ROI > 30% в segment analysis -- это малые выборки (n<100), не используются для принятия решений
\end{itemize}

\subsubsection{Стабильность}
\begin{itemize}
  \item Val-selected threshold (0.60) стабилен: coarse grid дает его в 8 из 8 экспериментов
  \item Fine val grid (0.01 step) переобучается: выбирает t=0.64 с ROI=6.81% (хуже)
  \item ROI на test устойчив: 7.23% +/- 0.03% через 8 итераций Phase 4
  \item Однако test set -- всего 3 дня. Долгосрочная стабильность не проверена
\end{itemize}

\subsubsection{Ограничения}
\begin{itemize}
  \item Test window: 2026-02-20 -- 2026-02-22 (3 дня) -- слишком короткий для надежной оценки
  \item ML_Team_Stats_Found всегда 'f' -- данные ELO/рейтингов не связаны со ставками
  \item ML_Winrate_Diff, ML_Rating_Diff: 100% NaN -- бесполезны
\end{itemize}

\subsection{What Didn't Work}

1. \textbf{Target encoding} (sport/market winrate): утечка через target statistics, доминирует в importance но не генерализует
2. \textbf{Weighted ensemble}: оптимизация весов на val переобучается (CB=0.55/LGB=0.05/XGB=0.40 дал t=0.85 -> ROI=-0.36%)
3. \textbf{Stacking (LogReg meta-learner)}: 1.89% -- хуже простого среднего
4. \textbf{EV-based selection}: val ROI=60.82% vs test ROI=-27.65% -- катастрофическое переобучение
5. \textbf{Per-odds-range thresholds}: overfit на val, test ROI=1.77%
6. \textbf{Training only on good sports}: меньше данных = хуже обобщение (ROI=2.26%)
7. \textbf{Deeper CatBoost (depth=5)}: early stop iter 59, ROI=2.23% -- overfitting
8. \textbf{Temporal features}: hour/day_of_week -- шум, не помогают
9. \textbf{Isotonic regression calibration}: маргинальный эффект на AUC (+0.0001), не влияет на ROI при val threshold
