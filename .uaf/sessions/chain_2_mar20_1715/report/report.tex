Без ML ставки дают -3.07\% (все ставки проигрывают маржу букмекера). Rule-based подход ещё хуже. Первый положительный ROI -- LogisticRegression с порогом 0.81. CatBoost default даёт 2.48\%, но останавливается на 19 итерациях из-за early stopping -- данные слишком зашумлены для глубокой модели без информативных фичей.

Ключевое наблюдение из Phase 1: feature importance CatBoost показал 85\%+ на Odds/implied\_prob. Модель по сути прогнозирует по структуре коэффициентов, а не по содержательным спортивным данным.

\subsection{Feature Engineering Results}

\subsubsection{Chain\_1 features (proven safe, Shadow Feature Trick)}
10 фичей, принятых через shadow feature trick в chain\_1 (delta > 0.002):
\begin{itemize}
  \item \texttt{log\_odds}, \texttt{implied\_prob}, \texttt{value\_ratio} -- трансформации коэффициентов
  \item \texttt{edge\_x\_ev}, \texttt{edge\_abs}, \texttt{ev\_positive}, \texttt{model\_implied\_diff} -- взаимодействия Edge/EV
  \item \texttt{log\_usd}, \texttt{log\_usd\_per\_outcome}, \texttt{parlay\_complexity} -- размер ставки и парлай
\end{itemize}

\subsubsection{ELO features (новое в chain\_2)}
15 safe ELO-фичей из \texttt{elo\_history.csv} и \texttt{teams.csv}:
\begin{itemize}
  \item Рейтинговые: \texttt{team\_elo\_mean}, \texttt{elo\_diff}, \texttt{elo\_spread}, \texttt{elo\_mean\_vs\_1500}
  \item Виннрейт: \texttt{team\_winrate\_mean}, \texttt{team\_winrate\_diff}
  \item Объем: \texttt{team\_total\_games\_mean}, \texttt{k\_factor\_mean}, \texttt{has\_elo}
\end{itemize}

\textbf{Leakage detection}: первоначальные ELO-фичи включали \texttt{ELO\_Change} (изменение рейтинга после матча) -- это target leakage, т.к. ELO\_Change зависит от результата матча. ROI с leakage был 25.90\%, после удаления -- 10.70\%.

\textbf{Эффект ELO-фичей:}
\begin{itemize}
  \item Все данные + safe ELO: ROI=2.38\% (vs 0.34\% baseline), +2.04 п.п.
  \item ELO-only subset: ROI=10.70\%, AUC=0.8540 -- прорывной результат
\end{itemize}

\textbf{Ограничение}: ELO-данные доступны только для 9.7\% ставок (7198 из 74493).

\subsection{Model Comparison}

Все модели обучены на ELO-only subset (ставки с данными ELO рейтингов):

| Модель | ROI | AUC | Threshold | N\_bets |
|--------|-----|-----|-----------|--------|
| CatBoost default | 10.70\% | 0.8540 | 0.62 | 725 |
| Optuna CatBoost | 16.63\% | 0.8431 | 0.73 | 634 |
| Optuna LightGBM | ~14\% | ~0.84 | - | - |
| XGBoost default | ~11\% | ~0.83 | - | - |
| Ensemble CB50 (CB 50\%/LGB 25\%/XGB 25\%) | \textbf{16.76\%} | \textbf{0.8501} | 0.62 | 743 |
| Rank average (CB+LGB+XGB) | 15.09\% | 0.84 | - | - |

Optuna дал значительное улучшение CatBoost (+5.93 п.п.). Основные гиперпараметры: depth=7, lr=0.214, l2\_leaf\_reg=1.15.

Ensemble CB50 -- лучший результат. Взвешенное среднее с доминированием CatBoost (50\%) стабильнее, чем equal average или rank average.

\begin{itemize}
  \item Threshold выбирается на val (последние 20\% train), применяется к test однократно

  \item Target encoding не используется
  \item Все фичи доступны до начала матча (pre-match)
  \item Максимальный ROI=16.76\% -- в пределах разумного (< 30\% sanity check)
\end{itemize}

\subsection{What Didn't Work}

\subsubsection{Chain\_1 (потолок 7.32\%)}
9 подходов Phase 4 chain\_1 дали маргинальный эффект (<0.1 п.п. каждый):
\begin{itemize}
  \item Weighted/stacking ensemble
  \item EV-based selection
  \item Per-odds-range thresholds
  \item Temporal features
  \item Deeper CatBoost
  \item Filtered training
  \item Fine val threshold
  \item Categorical Sport encoding
  \item Market/Tournament filtering
\end{itemize}

\textbf{Причина}: Odds/implied\_prob доминировали 85\% feature importance. Без внешних данных модель не могла выйти за пределы информации в коэффициентах.

\subsubsection{Chain\_2}
\begin{itemize}
  \item \textbf{Dual-model стратегия}: non-ELO component (~5\% ROI) размывает результат ELO-модели (16.76\%)
  \item \textbf{ELO coverage}: только 9.7\% ставок имеют ELO-данные. Это основное ограничение для масштабирования
  \item \textbf{Rank average ensemble}: чуть хуже weighted (15.09\% vs 16.76\%)
\end{itemize}
