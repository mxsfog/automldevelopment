# Executive Summary

## Цель

Предсказание победы ставки (won/lost) на спортивных событиях с целью достижения ROI >= 10% на отобранных ставках. Данные: платформа ставок, 20+ рынков, 10+ видов спорта.

## Лучший результат

- **Метрика ROI: 33.35%** (delta +4.77% vs baseline 28.58%)
- **Стратегия**: chain_8 CatBoost model (AUC=0.786) + Kelly threshold = p80 тренировочного распределения (0.5914 для LOW сегмента)
- **Объём ставок**: n=148 бет (Market=1x2, Soccer, pre-match, odds<1.8)
- **Фильтр**: Kelly criterion >= 80-й перцентиль тренировочного LOW-сегмента
- **Период теста**: Feb 20-22, 2026

## Ключевые выводы

- **Метод выбора порога без leakage**: валидационный период (Feb 17-20) имеет аномальный ROI=106% — любая оптимизация на val приводит к overfitting. Решение: выбор порога как перцентиль тренировочного Kelly-распределения полностью обходит проблему.
- **Что сработало**: повышение Kelly threshold от baseline 0.475 до p80=0.5914 выбирает top-20% бетов по уверенности модели → ROI +4.77%.
- **Главное ограничение**: платформенные ML-признаки (ML_Team_Stats, ML_Winrate_Diff) отсутствуют для всех релевантных бетов; ELO-данные покрывают <2%. Модель работает только на ценовых и временных признаках.

## Рекомендации

1. **Deployed стратегия**: `Market=1x2 AND lead_hours>0 AND Kelly >= 0.591 (p80 of train LOW Kelly)`
2. **Bet sizing**: half-Kelly для управления дисперсией при n~150 ставок/период
3. **Monitoring**: rolling 30-day ROI с alert при ROI < 0%
4. **Переобучение**: не чаще 1 раза в квартал; пересчитывать p80 threshold на новых тренировочных данных
5. **Следующий шаг для улучшения**: добавить внешние данные о командах (лиговые статистики, форма, травмы) — это может предоставить сигнал, которого нет в текущих признаках
6. **p82 threshold (0.6191)**: даёт ROI=36.02% n=127 — temporal analysis подтверждает genuine signal. Рассмотреть как новый baseline в следующей сессии при подтверждении на новом тестовом периоде.
